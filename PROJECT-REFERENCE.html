Structura fișierelor
Ce conține ZIP-ul și rolul fiecărui fișier

ZIP descărcat: core-recruit-netlify-cms.zip. Toate fișierele trebuie să fie la rădăcina repository-ului GitHub — nu într-un subfolder.

repository GitHub (rădăcină)/ ├── admin/ │ ├── index.html ← panoul de administrare /admin │ └── config.yml ← ce secțiuni apar în CMS (job-uri, echipă etc.) ├── content/ │ ├── jobs/ ← fiecare job = un fișier .md │ │ ├── marketing-lead.md │ │ └── senior-developer.md │ ├── team/ ← fiecare membru echipă = un fișier .md │ │ ├── cristina-jurconi.md │ │ ├── membru-2.md ← placeholder, editează din /admin │ │ └── membru-3.md ← placeholder, editează din /admin │ └── settings/ │ ├── contact.json ← email, telefon, adresă, social links │ └── homepage.json ← texte hero, statistici ├── css/ │ └── styles.css ← TOATE stilurile (culori, fonturi, layout) ├── js/ │ └── main.js ← meniu mobil, fade-in animații, CMS loader ├── images/uploads/ ← pozele încărcate din /admin se salvează aici ├── index.html ← Pagina principală ├── about.html ← Despre noi (echipă încărcată din CMS) ├── services.html ← Servicii (Recrutare, Team-building, Training) ├── careers.html ← Cariere (job-urile încărcate din CMS) ├── partners.html ← Parteneri și toate testimonialele ├── contact.html ← Contact cu formular EmailJS ├── maintenance.html ← Pagina "În curând" pentru mentenanță ├── netlify.toml ← configurație Netlify (redirects) └── SETUP-GUIDE.html ← ghid setup tehnic detaliat

05
GitHub — ghid pas cu pas
Cum actualizezi fișierele site-ului

De ce GitHub

GitHub stochează toate fișierele site-ului. Netlify preia automat orice modificare de pe GitHub și redesployează site-ul. Netlify CMS salvează conținutul (job-uri, echipă etc.) tot în GitHub. Fără GitHub, CMS-ul nu are unde salva datele.

Cum actualizezi un fișier HTML (text, structură)

1

Editează fișierul pe calculatorDeschide fișierul HTML în VS Code sau orice editor text și fă modificarea dorită

2

Copiază fișierul în folderul clonatÎnlocuiește fișierul vechi din folderul repository-ului local cu cel nou

3

Deschide GitHub DesktopVei vedea automat fișierele modificate listate în panoul din stânga

4

Scrie în câmpul Summary (câmpul MIC de sus)Ex: "update servicii page" — orice text scurt. NU în câmpul mare de jos (Description)

5

Click "Commit to main"Butonul devine activ doar după ce Summary-ul e completat

6

Click "Push origin"Apare în bara de sus după commit. Site-ul se actualizează automat în ~30 secunde

Problemă întâlnită: Butonul "Commit" nu se activa. Cauza: textul era scris în câmpul mare (Description) în loc de câmpul mic (Summary). Soluție: click în prima casetă mică, de o singură linie, deasupra casetei mari.

Cum adaugi foldere noi pe GitHub

Prin browser (github.com) nu poți uploada foldere — suportă doar fișiere individuale și nu păstrează structura de directoare. Folosește întotdeauna GitHub Desktop pentru foldere.

Problemă întâlnită: Când s-au uploadat fișierele prin github.com, folderele css/, js/, admin/, content/ nu s-au uploadat — doar fișierele HTML individuale. Asta a cauzat lipsa stilurilor pe site. Soluția: GitHub Desktop → drag & drop foldere → commit + push.

Setup inițial GitHub (o singură dată)

  1. Mergi la github.com → Sign up (gratuit)
  2. New repository → numește-l core-recruit → Public → Create
  3. Descarcă GitHub Desktop de la desktop.github.com
  4. File → Clone repository → selectează core-recruit → alege folder local
  5. Copiază toate fișierele din ZIP în folderul clonat
  6. GitHub Desktop → Summary: "initial upload" → Commit → Push origin

Netlify — setup & deploy
Hosting, Identity, domeniu

Deploy inițial

  1. Mergi la netlify.com → Sign up with GitHub
  2. Add new site → Import an existing project → GitHub → selectează core-recruit
  3. Lasă setările de build goale (site static, fără build command)
  4. Deploy site → live în ~30 secunde la xyz.netlify.app

Activare Netlify Identity (NECESAR pentru /admin)

1

Netlify Dashboard → Site configuration → Identity → Enable Identity

2

Registration → schimbă la "Invite only"IMPORTANT pentru securitate — altfel oricine poate accesa /admin

3

External providers → activează GitHub

4

Git Gateway → Enable Git GatewayPermite CMS-ului să scrie fișiere în GitHub

5

Identity → Invite users → office@core-recruit.roVei primi email de invitație — click link → setează parola

6

Mergi la yoursite.netlify.app/adminAutentifică-te → panoul CMS e activ

Actualizare config.yml (NECESAR)

Deschide fișierul admin/config.yml pe GitHub și modifică linia:

backend: name: github repo: USERNAME-TAU-GITHUB/core-recruit ← înlocuiește cu username-ul tău real branch: main

Cum folosești /admin
Ghid zilnic pentru administrare conținut

Accesează core-recruit.ro/admin din orice browser. Autentifică-te cu emailul și parola Netlify Identity.

Cum adaugi un job nou

1

Admin → Poziții deschise → + Poziție nouă

2

Completează câmpurile obligatorii:Titlu · Companie · Locație · Tip (Full-time / Hybrid etc.) · Categorie · Status · Link aplicare · Descriere scurtă

3

Link aplicare recomandat:mailto:office@core-recruit.ro?subject=Aplicatie+Titlu+Pozitie

4

Click "Publică" → apare pe site în ~30 secunde

Cum închizi o poziție: Editează job-ul → Status → "Închis" → Publică. Nu mai apare pe site dar rămâne salvat în sistem.
Cum marchezi ca urgent: Editează job-ul → Status → "Urgent" → Publică. Apare cu badge roșu "Urgent" pe site.
Important după publicarea unui job nou: Deschide fișierul careers.html, găsește array-ul const JOB_SLUGS = [...] și adaugă slug-ul noului job (ex: 'hr-manager'). Altfel job-ul e salvat în GitHub dar nu e încărcat pe pagina Cariere.

Cum editezi membrii echipei

1

Admin → Echipa → click pe "Nume Prenume" (membru-2 sau membru-3)

2

Completează: Nume, Rol, Biografie, LinkedIn URL, Ordine afișare

3

Upload fotografie: Click "Choose an image" → Upload → selectează poza → Choose selected. Formate: JPG, PNG, WebP. Max 2MB. Dimensiune recomandată: 800×600px

4

Publică → apare automat pe pagina Despre noi

Formular contact — EmailJS
Cum activezi trimiterea emailurilor

Formularul de contact din contact.html trimite emailuri la office@core-recruit.ro prin serviciul EmailJS. Gratuit până la 200 emailuri/lună.

1

Creează cont gratuit la emailjs.com

2

Email Services → Add New Service → GmailConectează-te cu office@core-recruit.ro · Copiază Service ID

3

Email Templates → Create New TemplateFolosește variabilele: {{from_name}}, {{from_email}}, {{company}}, {{phone}}, {{service}}, {{message}}. Setează "To email": office@core-recruit.ro. Copiază Template ID

4

Account → Public KeyCopiază Public Key

5

Deschide contact.html și caută:const EMAILJS_PUBLIC_KEY = 'YOUR_EMAILJS_PUBLIC_KEY';Înlocuiește cele 3 valori cu datele copiate din EmailJS

6

Commit + Push prin GitHub DesktopFormularul e activ imediat

Fără EmailJS configurat: Formularul deschide clientul de email local (Outlook, Gmail app) cu câmpurile pre-completate. Nu se pierd datele — utilizatorul trimite manual.

Conectare domeniu
core-recruit.ro → Netlify
Nu există downtime la switch! Site-ul vechi Wix rămâne live până în momentul în care DNS-ul se propagă complet. Nu ai nevoie de pagina de mentenanță pentru tranziție.
1

Netlify → Domain management → Add custom domainIntrodu core-recruit.ro

2

Netlify îți dă nameservereEx: dns1.p01.nsone.net · dns2.p01.nsone.net

3

Mergi la registrar-ul domeniuluiUnde ai cumpărat core-recruit.ro → DNS / Nameservers Settings → înlocuiește nameserver-ele existente cu cele de la Netlify

4

Așteaptă 1–24 orePropagarea DNS. SSL/HTTPS se activează automat și gratuit de către Netlify.

Pagina de mentenanță
Cum activezi / dezactivezi maintenance.html

Fișierul maintenance.html conține o pagină "Revenim în curând" în același stil cu site-ul — cu date de contact, progress bar animat și formular de notificare email.

Cum activezi pagina de mentenanță

1

În folderul clonat, redenumește index.html → index-backup.html

2

Redenumește maintenance.html → index.html

3

GitHub Desktop → Commit → Push → site-ul arată pagina de mentenanță

Cum dezactivezi (revii la site normal)

1

Redenumește index.html (mentenanță) înapoi la maintenance.html

2

Redenumește index-backup.html înapoi la index.html

3

GitHub Desktop → Commit → Push → site normal e live

Cum modifici progresul afișat

Deschide maintenance.html și caută:

  • width: 75% pe clasa .progress-fill → schimbă procentul
  • Secțiunea .status-list → editează textele și clasele (done, active, pending)
